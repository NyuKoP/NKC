# NKC v0.4.0 - Desktop UI and Tor-Only Routing

## Highlights

- Reworked account creation, start-key login, conversation, friend, settings, and profile surfaces around a consistent NKC desktop layout.
- Added stable profile colors keyed by profile or conversation identity so renaming a contact no longer changes the avatar background.
- Aligned profile avatars across the sidebar, conversation header, message timeline, and information panel.
- Added friend status messages, clearer friend and conversation navigation, improved section labels, and refined custom line icons.
- Improved message composition with responsive wrapping, compact bubble width limits, an auto-growing input, and smoother focus feedback.
- Improved light/dark theme transitions and applied the NKC semantic color palette consistently across controls, selections, toggles, and message bubbles.
- Persisted unlocked sessions through Electron's operating-system-backed encrypted secret store so the app can restore the signed-in account after a normal restart.
- Removed the retired alternate transport from settings, IPC, runtime lifecycle, installers, route policy, and related tests.
- Kept external Onion routing Tor-only and retained built-in Onion and Direct P2P modes.
- Added workspace CSS-language settings for Tailwind directives and required WebKit line-clamp declarations.

## Security and compatibility

- Session vault keys are stored only through Electron `safeStorage`; plaintext local or session storage is not used.
- Explicit lock, logout, reset, and wipe flows continue to clear the persisted session.
- The Tor component still requires a bundled pinned SHA-256 digest before installation or update.
- Existing locally encrypted account, contact, conversation, and message data remain in place during the upgrade.
