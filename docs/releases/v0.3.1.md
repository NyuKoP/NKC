# NKC v0.3.1 - Tor Reliability and Privacy

## Highlights

- Wired live Tor tests to the production Go transport worker and made the worker build mandatory for live runs.
- Added coalesced Tor route prewarming with a short successful-route cache that is invalidated on proxy changes.
- Classified proxy, SOCKS handshake, onion descriptor, service refusal, and upstream response failures with stable codes.
- Added bounded offline-queue retries with persisted attempt counts, exponential backoff, permanent failure status, and retry caps.
- Added coalesced Tor proxy recovery so queued messages remain safe while a failed proxy is restarted once.
- Redacted friend codes, onion addresses, IP addresses, device/contact identifiers, credentials, keys, message contents, paths, and error stacks at diagnostic and direct-console boundaries.
- Removed onion targets and device identifiers from Go forwarding trace events.

## Performance measurements

- Real Tor friend-code request and acceptance verification passed in both directions.
- Real Tor 32 KiB file delivery completed in 1.435 seconds and reverse chat delivery completed in 0.888 seconds after route readiness.
- Real Tor 10 MiB interruption/resume transfer completed in 121.320 seconds (0.082 MiB/s); chunk P50/P95 were 1.465/1.775 seconds and an in-transfer reverse chat arrived in 0.901 seconds.
- Real Tor 100 MiB interruption/resume transfer completed in 1,847.475 seconds (0.054 MiB/s); chunk P50/P95 were 2.038/2.917 seconds, an in-transfer reverse chat arrived in 4.606 seconds, and sender/receiver SHA-256 digests matched.
- The 500 MiB bounded-chunk benchmark completed with SHA-256-equivalent byte accounting:
  - legacy Base64 pipeline: 21.467 seconds, 163 MiB RSS growth, 666.8 MiB wire size;
  - binary pipeline: 0.232 seconds, 31 MiB RSS growth, 500.1 MiB wire size.
