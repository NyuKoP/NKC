# NKC v0.3.2 - Tor 15.0.18 and Login Safety

## Highlights

- Added official SHA-256 pins for every Tor Expert Bundle 15.0.18 platform asset.
- Added a focused regression check for the Windows x64 Tor 15.0.18 bundle.
- Prevented packaged builds from using a development renderer or the `nkc-electron-dev` data directory, even when development environment variables are present.
- Stopped persisting the unlocked vault key between app launches.
- Removed legacy persisted sessions on startup so an existing account requires its PIN or start key after relaunch.
- Added six-hour Tor update checks that automatically apply only newer releases with a trusted bundled hash.
- Verified Tor uninstall cleanup removes all installed versions and Tor download remnants while preserving hidden-service identity data.
- Restarted an active onion runtime after a successful component update and invalidated stale Tor binary paths.
- Routed explicit friend request, acceptance, and decline endpoints through the onion controller so first-contact responses do not depend on an established self-onion friendship route.
- Persisted accepted friend codes and refreshed the peer display name, status, avatar, and routing hints on acceptance.
- Increased the bounded file chunk size from 128 KiB to 1 MiB and aligned Electron/renderer/Go request and queue ceilings at 2 MiB.
- Removed security-sensitive `Math.random` fallbacks; identifier and routing-key creation now fails closed when secure randomness is unavailable.

## Large transfer measurements

- The optimized real Tor 10 MiB interruption/resume test completed in 73.998 seconds at 0.135 MiB/s, compared with 468.838 seconds at 0.021 MiB/s using 128 KiB chunks.
- The full real Tor 500 MiB transfer completed in 3,768.472 seconds at 0.133 MiB/s using 500 one-MiB chunks.
- Chunk P50/P95 were 6.780/10.834 seconds; reverse chat completed in 4.168 seconds during transfer.
- The intentional proxy interruption recovered, duplicate deliveries were zero, and sender/receiver SHA-256 digests matched.
