# NKC v0.3.2 - Tor 15.0.18 and Login Safety

## Highlights

- Added official SHA-256 pins for every Tor Expert Bundle 15.0.18 platform asset.
- Added a focused regression check for the Windows x64 Tor 15.0.18 bundle.
- Prevented packaged builds from using a development renderer or the `nkc-electron-dev` data directory, even when development environment variables are present.
- Stopped persisting the unlocked vault key between app launches.
- Removed legacy persisted sessions on startup so an existing account requires its PIN or start key after relaunch.
- Added six-hour Tor update checks that automatically apply only newer releases with a trusted bundled hash.
- Verified Tor uninstall cleanup removes all installed versions and Tor download remnants while preserving hidden-service identity data.
- Restarted an active onion runtime after a successful component update and invalidated stale Tor binary paths.

## Validation

- Focused Tor pin, runtime-mode, and session tests
- TypeScript node and renderer typechecks
- Focused ESLint validation
- UTF-8 validation
- Production Windows installer build
- Live Tor 15.0.18 install, verified removal, and clean re-download/reinstall
