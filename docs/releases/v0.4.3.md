# NKC v0.4.3 - Tor Reliability and Secure Media

## Highlights

- Made Tor the recommended network choice and added an optional automatic Tor preparation setting for application startup.
- Improved friend-code readiness so codes are exposed only after the selected Tor route is usable.
- Reduced friend-request retry delays and strengthened request, response, handshake, identity-binding, freshness, and replay validation.
- Improved connection-state reporting so the interface does not claim to be connected before the SOCKS route is reachable.
- Added verified application update status, release-note display, and clearer current-version or update-available feedback.

## Media and performance

- Aligned sender and receiver support for 1 MiB media chunks and retained the 500 MiB attachment limit.
- Added allowlisted image and video MIME handling, file-signature validation, exact chunk geometry checks, authenticated manifest binding, and final byte-count validation.
- Rejected active or mismatched preview content and normalized misleading filenames and extensions before display or download.
- Limited profile and group images to 10 MiB and safe raster formats.
- Moved large routed envelopes from the native worker JSON header into the binary IPC body, eliminating the header-size failure on application-sized chunks.
- Added adaptive transfer flow control, bounded encrypted storage operations, transfer progress persistence, and responsive chat during file delivery.

## Network and security

- Removed retired route configuration and runtime surfaces; external Onion routing now uses Tor only.
- Added signed internal Onion control messages and stricter route, device, nonce, and acknowledgement validation.
- Kept media encrypted at rest and authenticated in transit without sending attachments to an external inspection service.
- Made native forwarding failures return a bounded error response instead of leaving the local request pending.

## Validation

- Real Tor friend request and acceptance with protocol and signature verification.
- Bidirectional Tor file and chat delivery with matching SHA-256 values.
- Encrypted 1 MiB media transfer with interruption recovery, concurrent chat, and zero duplicate chunks.
- Friend-add UI coverage for Tor and default self-onion startup.
- Full TypeScript, Go, lint, unit, UI, production-build, and whitespace validation.
