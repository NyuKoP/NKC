# NKC v0.4.1 - Network Readiness and File Transfer Foundations

Release date: 2026-07-22

## Highlights

- Report a Tor connection only after the local SOCKS proxy is reachable, and publish a friend code only after the selected network is verified ready.
- Added independent Tor process lanes with separate data directories and SOCKS ports, round-robin selection, and per-lane route prewarming.
- Retained 1 MiB logical file chunks after live comparisons showed that smaller chunks add costly Tor request round trips.
- Added a dedicated unordered WebRTC file DataChannel while keeping ordinary chat traffic on its ordered channel.
- Added XChaCha20-Poly1305 file-transfer material derivation, fixed 96-byte AAD encoding, deterministic chunk nonces, bounded unordered frame assembly, and denial-of-service limits.
- Added a Go native receive store with random `WriteAt`, durable bitmap checkpoints, full-file SHA-256 verification, atomic finalization, and safe abort cleanup.
- Added Windows, Linux, and macOS preallocation implementations with safe fallback behavior.
- Added trusted Electron IPC methods for native receive initialization, chunk writes, checkpoints, finalization, and abort.

## Transfer policy and measurements

- Keep the logical plaintext chunk at 1 MiB.
- Use one Tor lane for photos and files below 200 MiB.
- Reserve two independent Tor lanes for photos and files of at least 200 MiB.
- Do not use three lanes by default; the measured 100 MiB run was slower and less stable.

A 500 MiB two-lane follow-up was stopped manually before completion and is not reported as a successful measurement.

## Security and compatibility

- Friend codes are withheld while Tor is merely starting or its SOCKS endpoint is unreachable.
- File chunks use independent authenticated encryption and strict UUID, length, index, frame-count, overlap, and memory bounds.
- Durable acknowledgement state is advanced only after file data and the bitmap journal have been synchronized.
- `SetFileValidData` is not used on Windows.
- Existing encrypted IndexedDB media remains compatible while the native receive pipeline is introduced incrementally.
